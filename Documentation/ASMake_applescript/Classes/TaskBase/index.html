_buildScript

Builds a script, script bundle, or applet. 

on _buildScript(
    sourceURL,
    encoding,
    fromURL,
    bundleURL,
    languageInstance,
    storageType,
    storageOptions)
Parameters
source

[NSString] or [text] The script's source code.

encoding

[boolean] The encoding the source code. When this is a missing value, the encoding is guessed.

fromURL

[NSURL] URL argument used to indicate the origin of scripts. This URL may be used to specify the location of the script libraries this script depends upon.

bundleURL

[NSURL] The location where the compiled script should be saved. It is assumed that this URL has the correct path extension (.scptd for script bundles and .app for applets).

storageType

A constant specifying the type of object to create: one of OSAStorageScriptType, OSAStorageScriptBundleType, OSAStorageApplicationType, OSAStorageApplicationBundleType or OSAStorageTextType.

languageInstance

[OSALanguageInstance] The language instance to be used to compile the script.

storageOptions

One ore more of OSANull, OSAPreventGetSource, OSACompileIntoContext, OSADontSetScriptLocation, OSAStayOpenApplet, or OSAShowStartupScreen.

Return Value

Nothing.

Discussion

This handler builds a script bundle or an applet including resources, auxiliary scripts and script libraries from source code in a directory with a specific layout. (This handler may be used to compile a script into a .scpt file as a special case, but _compile() is recommended instead.)

The sourceURL must point to the main script source file. An optional Info.plist file and an optional Resources folder may be placed in the same directory as the the main script. The Resources folder may contain any required resources, including additional scripts (in a Scripts subfolder) and embedded script libraries (in a Script Libraries subfolder). Any scripts and script libraries in source form are assumed to be laid out in the same way and are compiled recursively. (Compiled scripts and script libraries are simply copied to the target bundle.)

For example, a source directory for an applet may have the following content: 

                        MyApp
                                Info.plist
                                MyApp.applescript
                                Resources
                                        applet.icns
                                        Readme.txt
                                        Scripts
                                                helper.applescript
                                        Script Libraries
                                                LibOne
                                                        LibOne.applescript
                                                        Resources
                                                                ...
                                                org.me
                                                        Info.plist
                                                        LibTwo.applescript
                                                        Resources
                                                                ...

After building the applet, you will get an .app folder with the following content: 

                        MyApp.app
                                Contents
                                        Info.plist
                                        Resources
                                                applet.icns
                                                Readme.txt
                                                Scripts
                                                        main.scpt
                                                        helper.scpt
                                                Script Libraries
                                                        LibOne.scptd
                                                        org.me
                                                                LibTwo.scptd

Note that if the name of a script (without extension) is equal to the name of the folder that contains that script, such folder is not copied to the target bundle (e.g., LibOne.scptd is created immediately under Script Libraries, while LibTwo.scptd is created inside org.me).

Throws

An error if the script cannot be compiled or if it cannot be written to disk.

Discussion

This handler uses ditto to copy directory hierarchies. See man ditto for further information. Note that ditto does not copy items the same way as cp does. In particular, ditto("foo", "bar") will copy the contents of directory foo into bar, whereas cp("foo", "bar") will copy foo itself into bar.

posixPath

Turns (almost) any path specification into a POSIX path. 

on posixPath(
    aPath)
Parameters
aPath

[text], [file], or [alias] A path.

Return Value

[text] A POSIX path.

Discussion

This handler receives a path, in text form or as a file object or an alias, and returns it as a (relative or absolute) POSIX path. It does not perform any glob expansion (cf. glob()). This handler generalizes AppleScript's POSIX path of: in particular, it can be safely applied to relative POSIX paths (for example, POSIX path of "." is "/./", but posixPath(".") is still ".").

See Also

posixPaths

shell

Executes a shell command. 

on shell for command given options:options : {},
privileges:privileges : false,
username:username : missing value,
|password|:pass : missing value,
out:out : "",
err:err : "",
alteringLineEndings:ale : true
Parameters
command

[text] The command to be executed.

options

[text] or [list] An argument or a list of arguments for the command (internally, the handler always converts this to a list). Each option is quoted before the command is executed.

privileges

[boolean] A flag indicating whether the command should be executed as a different user. The default is false.

username

[text] The username that should execute the command. This argument is ignored unless privileges is set to true.

pass

[text] The password to be authenticated as username.

out

[text] Redirect the standard output to the specified file.

err

[text] Redirect the standard error to the specified file. Pass &1 to redirect to the standard output.

ale

[boolean] Whether line endings should be changed or not. The default is true.

Return Value

[text] The output of the command. If ASMake is run with --dry, returns the text of the command.

Discussion

This handler provides an interface to run a shell script via do shell script. All the features of do shell script are supported, including running a command with administrator privileges. For example, a command run as nick with password xyz123 may look as follows: 

                        shell for "mycmd" with privileges given options:{"--some", "--options"},
                                username:"nick", |password|:"xyz123"

Output redirection is supported, too, as well as altering line endings, e.g.: 

                        shell for "mycmd" given out:"/some/file", err:"&1", alteringLineEndings:false

This handler uses the syntax introduced is OS X 10.10 (Yosemite) for optional labeled parameters. Apart from the for parameter, all other arguments are optional (and may appear in any order).

Throws

An error if the shell script exits with non-zero status. The error number is the exit status of the command.
